## Архитектурные решения (НЕ МЕНЯТЬ без веской причины)

1. **Один движок, две точки входа.** `runPipelinePreset()` в `fincher.ts` вызывается и из Breakdown Studio и из StoryboardPanel. Нет отдельного "продакшен" пайплайна.

2. **ForEach = per-shot.** Если модуль зависит от результата с `shots[]` → запускается по одному на шот. Batch=1 для Gemini (rate limit).

3. **Промпт из пайплайна приоритетнее.** `shot.imagePrompt || buildImagePrompt(shot)` — если пайплайн создал промпт, старый promptBuilder не трогает.

4. **`buildScenePromptDrafts` пропускается** когда `activePipelinePreset` с модулями. Иначе он перезаписывает промпты из пайплайна.

5. **Пресеты самодостаточные.** Каждый `ProductionPreset` содержит стиль + режиссёрский промпт + модули пайплайна. Один клик в Bible → Director активирует всё.

6. **Адаптивные промпты.** Prompt Composer проверяет `hasRefImage` для каждого персонажа/пропа и меняет детализацию.

7. **LLM подбирает референсы.** Gemini Flash выбирает какие картинки из Bible отправить как refs. Пропсы приоритетнее персонажей.

8. **Все картинки в IndexedDB.** Bible images + шоты через `fileStorage.ts`. Переживает перезагрузку.

## Как работает цепочка (Анимация пресет)

```
1. Bible → Director → клик "Анимация"
   → activePipelinePreset = { modules: [Shot Planner, Prompt Composer] }
   → projectStyle = "3D animation, vibrant saturated colors..."
   → directorProfile = анимационный промпт

2. STORYBOARD → RE-BREAKDOWN на сцене
   → StoryboardPanel → breakdownSceneFincher(text, { bible, style })
   → fincher.ts видит activePipelinePreset → runPipelinePreset()

3. runPipelinePreset():
   → Module 1: Shot Planner (Gemini Flash, 1 вызов)
     Input: scene + style + characters + locations + props
     Output: { shots: [{ id, title, shotSize, visibleCharacters, visibleProps, propPlacement... }] }
   
   → Module 2: Prompt Composer (ForEach: N вызовов, по 1 на шот)
     Input: current_item (один шот) + all context
     Output: { shotId, imagePrompt, directorNote, cameraNote }

4. Результат → JenkinsShot[] с imagePrompt из пайплайна
   → buildScenePromptDrafts ПРОПУЩЕН (pipeline active)
   → промпт сохранён как есть

5. Генерация картинки:
   → LLM ref picker (Gemini Flash): "вот промпт, вот каталог Bible — какие refs?"
   → Refs → data URL → отправка в Nano Banana 2 + промпт
   → Результат в UI
```
