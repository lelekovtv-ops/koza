# KOZA Production Pipeline — Видение

## Концепция

KOZA Production — это операционная система кинопроизводства, которая связывает творческий процесс (сценарий → раскадровка) с производственным (кто, что, когда, какой техникой).

Ни один существующий инструмент не объединяет эти два мира. Сейчас команды используют WhatsApp для согласований, Google Docs для call sheets, Excel для графиков, и PDF-раскадровки по почте.

KOZA заменяет всё это единой платформой, где каждый видит только то, что ему нужно, и в том формате, который ему удобен.

## Архитектура

```
KOZA
├── Studio (творческий блок)
│   ├── Screenplay Editor      — написание сценария
│   ├── Breakdown Pipeline     — AI-раскладка на кадры
│   ├── Storyboard + Prompt Lab — визуализация + промпты
│   ├── Timeline               — монтажный таймлайн
│   └── Bible                  — персонажи, локации, стиль
│
├── Production (производственный блок)
│   ├── Call Sheets            — автогенерация из шотов
│   ├── Task Board             — факт-чекинг по сценам
│   ├── Role Views             — каждая роль видит своё
│   ├── Share Links            — доступ по ссылке
│   ├── Approval Flow          — согласования
│   └── Bots                   — AI-ассистенты по ролям
│
└── Infrastructure
    ├── Supabase               — мультиюзер, auth, storage
    ├── Real-time sync         — совместная работа
    └── Notifications          — уведомления команде
```

## Role Views — Что видит каждая роль

### Оператор
- Полная раскадровка в порядке съёмки (не сюжетном!)
- На каждый кадр: размер плана, движение камеры, объектив, свет
- Техника: какая камера, какой объектив, рельсы/кран/стедикам
- Референсы из Bible — как должна выглядеть локация
- Схема расстановки техники (будущее)

### Художник-постановщик
- Локации: полные описания + референсные изображения
- Реквизит: что нужно в каждой сцене (парсится из сценария)
- Костюмы: описания персонажей + визуальные якоря
- Палитра: цветовое решение из стиля проекта
- Декорации: что строить, что доставить

### Актёры
- Их сцены в хронологическом порядке
- Диалоги и действия из сценария
- Режиссёрские заметки к каждой сцене
- Эмоциональная карта (тон, переходы)
- Костюмы и грим для каждой сцены

### Продюсер
- Общий таймлайн производства
- Бюджет по сценам (будущее)
- Статус готовности: что снято, что нет
- Логистика: какие локации в какие дни
- Согласования: approve/reject на каждый этап

### Режиссёр (Jenkins)
- Всё вместе + AI-помощник
- Видение проекта из Bible
- Контроль над всеми решениями

## Task Board — Факт-чекинг

### Структура задач
```
Задача
├── Сцена (привязка)
├── Кадр (опционально)
├── Роль (оператор / художник / актёр / продюсер)
├── Статус (план → в работе → готово)
├── Исполнитель
├── Дедлайн
├── Комментарии
└── Фото-отчёт (с площадки)
```

### Автогенерация задач
Задачи создаются автоматически из данных брейкдауна:
- Каждый кадр → задача для оператора
- Каждая локация → задача для художника
- Каждый персонаж в сцене → задача для актёра
- Каждая сцена → задача согласования для продюсера

### Статусы
1. **План** — задача создана, ещё не начата
2. **В работе** — кто-то взялся
3. **На проверке** — сделано, ждёт approve
4. **Готово** — согласовано, закрыто

## Share Links — Доступ по ссылкам

### Принцип
Каждая роль получает свою уникальную ссылку:
```
koza.app/share/operator-abc123    → оператор видит раскадровку + технику
koza.app/share/art-def456         → художник видит локации + реквизит
koza.app/share/actor-ghi789       → актёр видит сцены + диалоги
koza.app/share/producer-jkl012    → продюсер видит всё + approve
```

### Уровни доступа
- **Read-only** — только просмотр
- **Comment** — просмотр + комментарии
- **Approve** — просмотр + согласование (approve/reject)
- **Edit** — полный доступ (для режиссёра)

### Реализация
- Supabase Row Level Security (RLS)
- JWT токены в ссылках
- Автоистечение ссылок (настраиваемое)

## AI Bots — Ассистенты по ролям

### Jenkins (уже есть)
- Главный режиссёрский ассистент
- Брейкдаун, анализ сцен, креативные решения
- Знает весь контекст проекта

### Operator Bot (будущее)
- Советует технику под задачу
- "Для этого кадра нужен 35mm, f/2.8, steadicam"
- Знает инвентарь техники (настраивается)
- Предупреждает о технических рисках

### Art Bot (будущее)
- Генерирует дополнительные референсы
- Составляет списки реквизита из сценария
- Палитры, мудборды, текстуры
- Предлагает решения для сложных локаций

### Producer Bot (будущее)
- Оптимизирует порядок съёмки (минимум переездов)
- Считает примерное время на каждую сцену
- Предупреждает о конфликтах в графике
- Генерирует отчёты для инвесторов

## Порядок съёмки vs Порядок сюжета

### Проблема
Сюжет идёт: Сцена 1 → 2 → 3 → 4 → 5
Но снимать эффективнее: 1, 3, 5 (одна локация) → 2, 4 (другая локация)

### Решение
KOZA автоматически группирует сцены по локациям для оптимального порядка съёмки:
```
День 1: Квартира Бориса
  - Сцена 1 (кадры 1-4)
  - Сцена 3 (кадры 8-11)
  - Сцена 5 (кадры 15-18)

День 2: Улица / Мост
  - Сцена 2 (кадры 5-7)
  - Сцена 4 (кадры 12-14)
```

Оператор получает раскадровку именно в этом порядке, а не в сюжетном.

## Конкурентное преимущество

Ни один продукт на рынке не даёт цепочку:
```
Сценарий → AI-раскадровка → Производственный план → Исполнение → Факт-чекинг
```

Ближайшие конкуренты:
- **StudioBinder** — call sheets и scheduling, но без AI и без раскадровки
- **ShotDeck** — референсы, но без production management
- **Celtx** — сценарий + breakdown, но без AI-генерации визуала
- **Frame.io** — ревью видео, но не pre-production

KOZA объединяет все этапы и добавляет AI на каждом шаге.
