## 1. Аудит и исправление багов

### Исправлено
- **Regex stack overflow** в `/api/nano-banana` и `/api/gpt-image` — base64 строки вызывали `Maximum call stack size exceeded`. Заменено на `indexOf`/`slice`.
- **Retry при 503** — добавлен автоматический retry (3 попытки с задержкой 3/6/9с) при перегрузке Google API.
- **Google API key** — `@ai-sdk/google` искал `GOOGLE_GENERATIVE_AI_API_KEY`, а в проекте `GOOGLE_API_KEY`. Исправлено через `createGoogleGenerativeAI({ apiKey })`.
- **Неиспользуемый параметр `allShots`** — убран из `buildImagePrompt()` и всех 11 вызовов в 6 файлах.
- **Время суток** — теперь берётся из `LocationEntry.timeOfDay` напрямую вместо regex-парсинга.

### Найдено но не критично
- `resetRun` в pipeline.ts — необычное форматирование, но валидный JS.
- Nano-banana модели — существуют как API route, не баг.

## 4. Prompt Censor

### Per-shot цензор
- Валидация якорей персонажей/локаций.
- Противоречия времени суток (ночная сцена + дневное освещение).
- Удаление дубликатов строк.
- Предупреждение о длине промпта > 3000 символов.

### Continuity Censor (cross-shot)
- Анализ всех шотов сцены вместе.
- Разрывы присутствия персонажей (есть в shot 1 и 4, нет в 2 и 3).
- Screen direction (правило 180°).
- Внезапное появление персонажа без входа.
- Отсутствие establishing shot.
- Per-shot директивы.

## 5. Оптимизация пайплайна

### Параллелизация
- Context Router + Creative Planner теперь через `Promise.all()`.

### Per-stage model routing
- Структурные стадии (Scene Analyst, Action Splitter, Context Router, Censor) → `gemini-2.5-flash`.
- Креативные стадии (Creative Planner, Shot Planner, Prompt Composer) → `gpt-4o`.

### Устранение bottleneck'ов
- Creative Planner: 30 filmHistoryReferences → 5 (было 39.5с).
- Prompt Composer: убраны `imagePackage` + `videoPackage` (19 лишних полей на шот, нигде не использовались).

### Настройки скорости
- Новые параметры в Dev Settings: Speed (Fast/Balanced/Quality), модели для креативных и структурных стадий.
- Сохраняются в localStorage, читаются jenkins.ts при каждом брейкдауне.

### Обязательные якоря
- Scene Analyst, Shot Planner, Prompt Composer — обязаны использовать точные имена из Bible.

### Результат
- **Было: ~100с** → **Стало: ~30-35с** (ускорение ~3x).
