## Что мы строим

**AI Production Studio** — платформа полного цикла кинопроизводства. Не редактор сценариев, не генератор картинок — а **единственный инструмент где весь процесс от идеи до съёмки живёт в одном месте.**

### Для кого
- Голливудские и независимые студии (сериалы, полный метр)
- Независимые режиссёры, продюсеры
- YouTube-каналы, Reels, короткий контент
- Все кто работает с видео и хочет оптимизировать время и бюджет

### Непрерывный воркфлоу (всё взаимосвязано)
```
Идея
  ↓
Сценарий (AI-ассистент пишет вместе с тобой)
  ↓
Автоматический breakdown → Сцены → Шоты
  ↓
Раскадровка (AI-генерация + реальные кадры с камеры)
  ↓
Препродакшен (документация, call sheets, бюджет, расписание)
  ↓
Продакшен (боты-ассистенты, планирование съёмок, команда)
  ↓
Постпродакшен (таймлайн, монтажные заметки, экспорт)
```

Каждый этап **автоматически питает следующий**. Написал сцену — она уже в раскадровке, в расписании, в breakdown.

## Главная архитектурная идея — Shot как единая сущность

**Раскадровка = Таймлайн = Стек нодов = Доска — это одна и та же сущность (Shot).**

Один шот — это одновременно:
- Визуальная карточка в раскадровке
- Позиция на таймлайне
- Нода на доске
- Строка в документации для съёмки

Пользователь не переключается между инструментами — он переключает **представление** одних и тех же данных.

### Реальные кадры + AI = единый процесс
```
AI-генерация шота (препродакшен)
        ↕  заменяется когда снято
Реальный файл с камеры (продакшен)
        ↕
Монтажная нода (постпродакшен)
```
Связь шота не рвётся на всём пути.

## Сценарий связан с нодами в реальном времени

```
Пишешь текст в редакторе
        ↓
Парсер определяет: INT. КУХНЯ — ДЕНЬ  → Scene #12
        ↓
Scene #12 автоматически появляется на доске как нода
        ↓
Дженкинс (AI-бот) предлагает breakdown:
  Shot 12A — крупный план
  Shot 12B — общий
  Shot 12C — реакция
        ↓
Пользователь видит в реальном времени:
  сцена 12 / ~4 шота / ~18 секунд экранного времени
```

## Дженкинс — AI-ассистент по раскадровке

Это не просто AI — это **личность**. Опытный оператор-постановщик с характером и мнением:
- Предлагает операторские решения исходя из текста сцены
- Знает бюджет и график — оптимизирует съёмку
- Говорит конкретно: *"Три шота можно объединить в один долгий план — сэкономишь день"*
- Умеет работать в схемах: shot-reverse-shot, голландский угол, долгий план

## Технический стек

- **Frontend:** Next.js 14 (App Router) + TypeScript + Tailwind CSS
- **State:** Zustand (persist)
- **AI SDK:** Vercel AI SDK (`ai`, `@ai-sdk/openai`, `@ai-sdk/anthropic`, `@ai-sdk/google`)
- **Image Gen:** `@google/genai` (Imagen 4 Fast), `openai` (DALL-E 3)
- **UI:** lucide-react, @radix-ui
- **Board:** ReactFlow (нодовый редактор)
- **Storage:** IndexedDB (бинарные файлы), Zustand persist (метаданные), Supabase (облако — планируется)
- **Путь проекта:** `~/Desktop/ai-aggregator-v2` ← ВСЕГДА использовать этот путь

## Правила для Claude (обязательно соблюдать)

1. **Путь проекта всегда** `~/Desktop/ai-aggregator-v2` — никогда не писать старый путь
2. **Каждую фичу анализировать через призму: нужно ли это пользователю?** Не "можно ли сделать", а "зачем пользователю и решает ли это его задачу"
3. **Не ломать существующий UI** — канва, редактор, переходы, темы
4. **Ключи хранения** начинаются с `koza-`
5. **Язык:** логика и общение на русском, код на английском
6. **Стиль UI:** минимализм, стеклянный эффект (backdrop-blur), кремово-светлая / тёмная тема
7. **Иконки:** только lucide-react
8. **Shot — единая сущность.** Раскадровка = Таймлайн = Нода = Доска. Никогда не делать их разными несвязанными сущностями
9. **Сценарий связан с нодами** в реальном времени — изменение сцены отражается на доске
10. **Дженкинс** — имя AI-ассистента по раскадровке. Это личность, не просто функция
