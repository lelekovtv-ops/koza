# Work Log — 2026-03-31 (night session)

## Screenplay Editor — Major Overhaul

### Парсер сценария — 8 фиксов

1. **Кириллица в scene heading** — `ИНТ./ЭКСТ.` теперь распознаются в `RE_SCENE_HEADING` и `getLiveTypeConversion`
2. **Character с возрастом** — `БОРИС (55)`, `МАРИНА (35 лет)`, `HR-МЕНЕДЖЕР` корректно определяются
3. **Dialogue chain** — одна пустая строка между character и dialogue не рвёт цепочку; 2+ строки = новая секция
4. **Русские transition/shot** — ЗАТЕМНЕНИЕ:, ПЕРЕХОД:, КРУПНЫЙ ПЛАН:, ДЕТАЛЬ: и т.д.
5. **Контекст при вставке** — `insertData` передаёт prevType текущего блока в `parseTextToBlocks`
6. **FADE IN: → transition** — regex `FADE IN:?` (опциональное двоеточие), + фильтр в `parseCharacters`
7. **Action сохраняет регистр** — убран `toSentenceCase`, авторский текст не искажается
8. **normalizeNode** — убрана агрессивная авто-конверсия action→dialogue (ломала Enter flow)

### Page Break Engine (новый файл)

`src/components/editor/screenplay/screenplayPageBreaks.ts` — 200 строк, индустриальные правила:

- **Orphan heading** — scene_heading не последний на странице
- **Character keep-together** — CHARACTER + первая строка DIALOGUE всегда вместе
- **Transition keep-with-next** — не отрывается от следующего scene_heading
- **(MORE) / (CONT'D)** — при разрыве диалога: визуальные маркеры в рендере
- **Widow protection** — минимум 2 строки на каждой стороне разрыва
- **Parenthetical protection** — не отрывается от character

### Ambient Focus Mode (`Cmd+F`)

`src/components/editor/screenplay/AmbientFocusMode.tsx`

**Концепция**: Полноэкранный режим где фон оживает иллюстрациями сцены пока пишешь.

- **Browser Fullscreen API** — убирает весь UI
- **Страница A4 как матовое стекло** — `backdrop-blur(6px)`, `bg-black/35`, видно картинку под текстом
- **По краям экрана** — яркая иллюстрация (opacity 0.55)
- **Comic book стиль** — bold ink, halftone dots, speed lines, sound effects (BOOM, CRASH), split panels
- **Progressive panels** — 1 img = splash page, 2 = cause/effect, 3 = strip, 4+ = grid
- **Rolling window** — хранит последние 6 картинок, старые заменяются новыми
- **Scene-based trigger** — генерирует при смене сцены или 100+ новых символов
- **Bible context** — персонажи и локации из Bible вплетаются в промпт
- **Persist** — картинки сохраняются в IndexedDB, per-project
- **Idle fade** — 60с без активности → сценарий исчезает (3с), видны только иллюстрации; касание → возврат (0.4с)
- **Model**: nano-banana (cheapest), cooldown 30с
